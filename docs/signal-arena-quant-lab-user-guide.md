# Signal Arena Quant Lab 使用指南（草稿）

日期：2026-06-12  
状态：草稿，正式实现后需要随页面和策略参数同步更新

## 1. 这份指南解决什么问题

Quant Lab 的目标不是让你只看一个“今天赚了还是亏了”的结果，而是让你能参与判断：

- 策略为什么买、为什么卖、为什么观望。
- 最近表现好不好，是运气、行情，还是规则真的有效。
- 哪些参数可能需要调整。
- 怎么让 Codex 帮你做调参实验，而不是直接盲改线上策略。

这仍然是模拟交易学习项目，不是投资建议。

## 2. 每天看什么

每天只需要先看四件事。

### 2.1 运行是否正常

看：

- 今日是否有 Runner 运行记录。
- 是否出现流程级失败。
- 是否因为市场关闭、上游限速、数据不足而跳过。

判断：

- 偶尔因为休市或数据不足跳过是正常的。
- 连续 3 个交易日流程级失败，需要先修系统，不讨论策略好坏。

### 2.2 收益曲线和回撤

看：

- 总资产是否持续创新高。
- 最大回撤是否扩大。
- 回撤后是否能恢复。

判断：

- 一天亏损不能说明策略坏。
- 回撤超过预警线，例如 `-8%`，需要进入防守观察。
- 回撤超过强制线，例如 `-12%`，应考虑冻结新开仓或回滚策略。

### 2.3 最近动作

看：

- 本轮是买入、卖出、观望，还是被风控拦截。
- 如果买入，候选分数是否足够高。
- 如果观望，是没有机会，还是策略过于保守。
- 如果卖出，是止损、止盈、趋势破坏，还是仓位再平衡。

判断：

- 有明确理由的观望是有效决策。
- 频繁“差一点买入”可能说明阈值过高。
- 频繁买入后很快卖出，可能说明追高惩罚或冷却期不够。

### 2.4 持仓是否健康

看：

- 单只股票仓位是否过高。
- 现金比例是否过低。
- 持仓数量是否过多。
- 是否有不可卖持仓受 T+1 限制。

判断：

- 现金过低会让策略没有防守空间。
- 持仓过多会让策略变成“什么都买一点”，难以归因。
- 持仓过集中会让单只股票决定整个账户表现。

## 3. 每周复盘看什么

每周至少看一次完整复盘。

核心指标：

- 总收益率：策略最终赚/亏多少。
- 最大回撤：最痛的时候亏到哪里。
- 交易次数：策略是否太频繁或太保守。
- 胜率：赚钱交易占比。
- 盈亏比：赚的时候能赚多少，亏的时候亏多少。
- 换手率：是否频繁换仓。
- 空仓率：有多少时间没有持仓或没有新动作。
- 风控拦截次数：策略是否经常给出不合法动作。

判断优先级：

1. 先看系统是否稳定。
2. 再看最大回撤是否可接受。
3. 再看收益。
4. 最后才看胜率。

胜率不是最重要的。一个策略可以胜率不高，但亏小赚大；也可以胜率高，但一次大亏吃掉全部收益。

## 4. 关键参数是什么意思

正式实现后，以页面展示和策略配置为准。下面是 `Q-Alpha v1` 计划中的核心参数。

### 4.1 买入分数阈值

作用：

- 决定候选股至少要拿到多少总分，策略才允许买入。
- 它是“开不开新仓”的总闸门。
- 计划中的初始值是 `70`，表示只有综合评分达到较高质量的候选才会进入买入队列。

调高会怎样：

- 买入更少，更挑剔。
- 空仓或现金过高的时间可能变长。
- 可以减少误买，但可能错过早期机会。

调低会怎样：

- 买入机会变多。
- 账户更容易满仓。
- 误买、追高、频繁交易的概率上升。

什么时候考虑调整：

- 如果连续 10 个交易日几乎不买入，而很多候选都稳定在 `65-69` 分，可以测试把阈值从 `70` 降到 `65`。
- 如果买入后经常很快亏损或被止损，可以测试把阈值从 `70` 提到 `75`。

常见误区：

- 不要因为错过一只大涨股就立刻降低阈值。先看一组样本，而不是一只股票。

### 4.2 卖出分数阈值

作用：

- 决定持仓分数跌到什么程度时，策略开始考虑卖出或减仓。
- 它不是单独的止损线，而是“这个持仓的综合质量是否变差”的判断。

调高会怎样：

- 更早卖出变弱的持仓。
- 回撤可能更小。
- 也更容易卖飞，也就是刚卖出后股票又涨。

调低会怎样：

- 给持仓更多波动空间。
- 可能减少频繁换仓。
- 但亏损扩大时反应会更慢。

什么时候考虑调整：

- 如果经常盈利回吐，说明退出可能太慢，可以提高卖出敏感度。
- 如果经常卖出后很快又重新走强，说明卖出可能太早，可以降低卖出敏感度或加强趋势确认。

常见误区：

- 卖出阈值不能只看单笔交易，要结合最大回撤和卖飞次数一起看。

### 4.3 趋势权重

作用：

- 衡量均线结构和趋势方向有多重要。
- 典型判断包括 `MA20 > MA60`、价格是否站上 `MA20`、均线是否向上。
- 趋势权重高，策略更相信“强者恒强”。

调高会怎样：

- 更偏向已经形成上升趋势的股票。
- 在趋势行情中可能更稳定。
- 在震荡行情中可能买得偏晚。

调低会怎样：

- 策略不那么依赖均线趋势。
- 可能更早发现刚启动的股票。
- 也更容易买到还没真正走强的股票。

什么时候考虑调整：

- 如果买入经常太晚，已经涨过一大段才进，可以测试降低趋势权重或提高追高惩罚。
- 如果买入很多“看起来便宜但继续跌”的股票，可以测试提高趋势权重。

常见误区：

- 趋势权重不是越高越好。太高会让策略只敢买已经很明显的走势，收益空间可能变小。

### 4.4 动量权重

作用：

- 衡量近期涨幅和相对强弱有多重要。
- 典型指标包括 `5D / 10D / 20D` 收益率。
- 动量权重高，策略更愿意买近期表现强的股票。

调高会怎样：

- 更容易追随强势股。
- 在强趋势行情中可能表现更好。
- 震荡行情里更容易追在短期高点。

调低会怎样：

- 不那么追近期涨幅。
- 交易会更稳一点。
- 可能错过快速拉升的机会。

什么时候考虑调整：

- 如果策略总是买入后短期回落，可以降低动量权重或提高追高惩罚。
- 如果策略长期买不到市场主线，可以提高动量权重。

常见误区：

- 动量不是“涨得越多越该买”。短期涨太多仍然要被追高惩罚约束。

### 4.5 突破权重

作用：

- 衡量股票是否正在突破近期高点或重要区间。
- 它帮助策略识别“价格开始进入新阶段”的候选。

调高会怎样：

- 更偏突破交易。
- 可能抓到趋势启动点。
- 假突破时容易买入后回落。

调低会怎样：

- 更少追突破。
- 更偏等待趋势确认或回撤机会。
- 可能错过快速突破行情。

什么时候考虑调整：

- 如果经常买到假突破，可以降低突破权重，或要求成交量确认。
- 如果总是等到趋势很明显才买，可以提高突破权重。

常见误区：

- 突破必须配合成交量和波动判断。单独突破不等于买入理由充足。

### 4.6 成交量确认权重

作用：

- 判断上涨或突破是否有成交量支持。
- 量能放大通常说明关注度上升，但极端放量也可能是短期过热。

调高会怎样：

- 更重视“有资金参与”的走势。
- 可以过滤一些无量上涨。
- 可能错过温和上涨的股票。

调低会怎样：

- 对成交量不那么敏感。
- 买入机会更多。
- 也更容易买到缺少资金确认的走势。

什么时候考虑调整：

- 如果策略买到很多无量上涨后回落的股票，可以提高成交量确认权重。
- 如果策略错过很多稳定慢涨股票，可以降低成交量确认权重。

常见误区：

- 放量不是越大越好。异常巨量可能意味着短线分歧很大，需要和追高惩罚一起看。

### 4.7 波动惩罚

作用：

- 对波动太大的股票扣分。
- 它用来控制账户曲线的剧烈起伏。

调高会怎样：

- 更排斥大起大落的股票。
- 最大回撤可能更小。
- 也可能错过高弹性机会。

调低会怎样：

- 更愿意参与高波动股票。
- 收益弹性可能变大。
- 账户回撤也可能明显变大。

什么时候考虑调整：

- 如果收益曲线像坐过山车，优先考虑提高波动惩罚。
- 如果策略过于保守，长期只买低波动但没弹性的股票，可以降低波动惩罚。

常见误区：

- 波动不是风险的全部。低波动股票也可能慢慢阴跌，所以还要看趋势和回撤。

### 4.8 追高惩罚

作用：

- 对短期涨太多、离均线太远、过热的候选扣分。
- 它用来避免策略在情绪最热的时候冲进去。

调高会怎样：

- 更不愿意追涨。
- 买点可能更稳。
- 可能错过强势连续上涨。

调低会怎样：

- 更敢追强势。
- 可能抓住主升段。
- 也更容易买在短期高点。

什么时候考虑调整：

- 如果买入后经常短期回撤，可以提高追高惩罚。
- 如果市场明显强趋势，而策略一直不敢买主线，可以降低追高惩罚或提高动量权重。

常见误区：

- 追高惩罚和动量权重是一对拉扯参数。不要只改一个却不观察另一个。

### 4.9 最大单股仓位

作用：

- 限制单只股票最多占总资产多少。
- 计划初始上限是 `20%`，意思是单只股票不能决定整个账户命运。

调高会怎样：

- 看对时收益更明显。
- 看错时回撤也更明显。
- 账户更集中。

调低会怎样：

- 风险更分散。
- 单笔错误伤害更小。
- 但收益也可能被摊薄。

什么时候考虑调整：

- 如果单只股票对账户影响太大，可以降低上限。
- 如果策略命中率高但收益被过度分散，可以测试提高上限。

常见误区：

- 仓位上限不是用来追求刺激的，它首先是防止一次错误毁掉整条曲线。

### 4.10 最低现金比例

作用：

- 要求账户始终保留一部分现金。
- 现金是防守空间，也是后续机会的弹药。

调高会怎样：

- 更保守。
- 回撤可能更小。
- 但牛市或强趋势行情中收益可能落后。

调低会怎样：

- 更激进。
- 更容易满仓。
- 一旦行情转弱，调整空间更小。

什么时候考虑调整：

- 如果账户经常满仓后遇到回撤，可以提高最低现金比例。
- 如果长期现金过高且错过大量高分机会，可以降低最低现金比例。

常见误区：

- 现金不是“没工作”。在量化系统里，现金本身就是一种防守仓位。

### 4.11 最大持仓数

作用：

- 限制最多同时持有几只股票。
- 它影响分散程度和策略归因难度。

调高会怎样：

- 更分散。
- 单只股票影响降低。
- 但持仓太多会让策略接近指数，难以看出规则是否有效。

调低会怎样：

- 更集中。
- 更容易看清每笔交易的贡献。
- 波动和单票风险更大。

什么时候考虑调整：

- 如果持仓太多、每只都很小、收益不明显，可以降低最大持仓数。
- 如果单票风险太大，可以提高最大持仓数或降低单股仓位上限。

常见误区：

- 分散不是越多越安全。过度分散会让策略失去锋芒，也更难复盘。

### 4.12 止损线

作用：

- 当持仓亏损或从高点回撤超过阈值时，触发卖出或减仓。
- 它是防止小亏变大亏的最后保护。

调高会怎样：

- 更能容忍波动。
- 减少被短期洗出去。
- 但真正走坏时损失更大。

调低会怎样：

- 更早止损。
- 单笔亏损更可控。
- 但容易被正常波动洗出。

什么时候考虑调整：

- 如果经常小亏变大亏，可以收紧止损线。
- 如果经常刚止损就反弹，可以放宽止损线，或增加趋势确认后再卖。

常见误区：

- 止损不是承认失败，而是让策略有资格继续下一笔交易。

### 4.13 卖出后冷却期

作用：

- 卖出某只股票后，短时间内不允许重新买入同一标的。
- 它用来避免“刚卖又买、刚买又卖”的来回交易。

调高会怎样：

- 降低反复横跳。
- 交易更稳定。
- 可能错过快速二次启动。

调低会怎样：

- 更快重新进入同一股票。
- 机会捕捉更灵活。
- 也更容易被震荡行情反复打脸。

什么时候考虑调整：

- 如果同一只股票短期反复买卖，可以提高冷却期。
- 如果卖出后经常很快重新走强，可以降低冷却期，但要先确认不是偶然样本。

常见误区：

- 冷却期不是为了惩罚某只股票，而是为了约束策略自己的冲动。

### 4.14 参数之间的关系

很多参数不能单独看：

- `动量权重` 和 `追高惩罚` 是一组。前者鼓励买强势，后者防止买过热。
- `买入分数阈值` 和 `最低现金比例` 是一组。阈值低、现金比例也低，会让策略很快满仓。
- `最大单股仓位` 和 `最大持仓数` 是一组。单股上限高、持仓数少，账户会更集中。
- `卖出分数阈值` 和 `止损线` 是一组。前者看综合质量，后者看亏损控制。
- `趋势权重` 和 `突破权重` 是一组。前者偏确认，后者偏启动。

调参时一次只改一类问题。例如只测试“买入阈值”，不要同时改仓位、止损和因子权重。否则就算结果变好，也不知道到底是谁带来的。

## 5. 什么时候应该考虑调参

可以考虑调参：

- 连续 10 个交易日长期空仓，且候选股经常接近买入阈值。
- 频繁买入后短期止损，说明买入太激进或追高惩罚不够。
- 经常盈利回吐，说明止盈保护或趋势退出太慢。
- 最大回撤持续扩大，说明仓位、现金比例或止损规则需要加强。
- 风控经常拦截同类动作，说明策略输出和交易规则没有对齐。

不应该立刻调参：

- 只因为某一天亏钱。
- 只因为错过了一只大涨股票。
- 只因为某一次买入后短期浮亏。
- 还没有足够运行样本。
- 系统本身还在失败或数据不完整。

## 6. 调参的正确流程

不要直接改线上主策略。正确流程是：

1. 记录现象。
2. 提出假设。
3. 只改一类参数，生成候选版本。
4. 用历史数据回测。
5. 影子运行一段时间，只记录不下单。
6. 比较主版本和候选版本。
7. 决定是否切换。

示例：

```text
现象：最近 10 个交易日几乎没有买入，但候选股经常在 65-69 分。
假设：买入阈值 70 过高。
实验：创建 v2-candidate，把买入阈值从 70 降到 65，其他参数不变。
验证：比较收益、最大回撤、交易次数、空仓率。
```

## 7. 怎么让 Codex 帮你调整

你可以直接复制这个模板：

```text
请分析最近 N 天 Signal Arena Quant Lab 的策略日志。

我观察到：
- 写你看到的现象。

我的假设是：
- 写你怀疑的参数或规则。

我想测试：
- 写具体参数变化，例如买入阈值 70 -> 65。

要求：
- 不要直接改线上主策略。
- 先创建候选版本。
- 做回测或影子运行对比。
- 输出收益、最大回撤、交易次数、空仓率、风控拦截次数。
- 最后告诉我是否建议切换。
```

如果你不知道该调什么，可以这样问：

```text
请你看最近两周的策略日志和收益曲线，找出最值得研究的 1-3 个策略问题。
不要改参数，先解释证据、可能原因、建议实验和风险。
```

## 8. 你可以参与哪些决策

你可以参与：

- 是否接受当前策略版本继续运行。
- 是否开启候选版本实验。
- 是否提高或降低风险偏好。
- 是否偏向少交易高确定性，还是多交易多机会。
- 是否更重视收益，还是更重视回撤控制。
- 是否把某个候选版本切换为主策略。

你不需要每天做：

- 手动判断每只股票。
- 手动下单。
- 逐条修改技术指标。
- 因为短期涨跌频繁干预。

## 9. 我开发时应该交付什么

Quant Lab 完成后，除了代码和页面，还应该交付：

- 当前策略版本说明。
- 当前参数表。
- 每个参数的人话解释。
- 每日查看指南。
- 每周复盘指南。
- 调参请求模板。
- 回测报告格式。
- 候选版本和主版本的切换规则。

如果页面上某个交易你看不懂，就说明系统解释还不够好，需要继续补页面和文档，而不是让你硬猜。
