# Roadmap: Tools Branch Evolution

## Overview

This roadmap expands `maysssss.cn` beyond the comic portal by growing the tools
branch in small public-safe steps. The first pass established the hub and QR
generator; the current pass adds a searchable UI style prompt index that stays
useful, visually calm, and easy to verify.

## Phases

- [x] **Phase 4: Tools Hub And Portal Entrance** - Add `/tools`, establish the
  holographic design system, and promote Tools to a live homepage panel.
- [x] **Phase 5: QR Studio Core Functionality** - Build the first usable public
  tool with editable QR generation and download flows.
- [x] **Phase 6: Accessibility, Polish, And Verification** - Refine responsive
  behavior, keyboard flow, and validation safety across the new tools branch.
- [x] **Phase 7: Style Prompt Index** - Add `/tools/style-prompt` with local
  prompt data, stable previews, and a simplified copy-focused browsing flow.

## Phase Details

### Phase 4: Tools Hub And Portal Entrance

**Goal**: The main site gains a readable tools hub and a real `Tools` entrance
without losing the current homepage composition.
**Depends on**: Phase 3
**Requirements**: HUB-01, HUB-02, HUB-03, PORT-01, PORT-02, PORT-03
**Success Criteria**:
1. User can enter `/tools` from the homepage through a real comic-panel link.
2. `/tools` immediately reads as a tools hub rather than a vague concept page.
3. The holographic styling feels distinctive but does not obscure names,
   descriptions, or actions.
4. One sealed future panel remains on the homepage, preserving continuation.
**Plans**: 2 plans

Plans:
- [x] 04-01: Design and implement the `/tools` hub layout, card system, and
  holographic visual language.
- [x] 04-02: Update the homepage portal grid to expose `Tools` while keeping the
  comic composition balanced.

### Phase 5: QR Studio Core Functionality

**Goal**: `QR Studio` becomes the first live tool, with clear generation,
customization, and download flows.
**Depends on**: Phase 4
**Requirements**: HUB-04, QR-01, QR-02, QR-03, QR-04, QR-05, UX-02
**Success Criteria**:
1. User can open the QR tool from `/tools` in one click.
2. User can generate a QR code from plain text or a URL without leaving the
   page.
3. User can change size, margin, and colors with immediate visible effect.
4. User can download the result as both PNG and SVG.
5. The tool stays frontend-only with no auth, uploads, or storage dependency.
**Plans**: 2 plans

Plans:
- [x] 05-01: Build the QR Studio editing interface and live QR output.
- [x] 05-02: Add PNG/SVG export, reset flow, and hub-to-tool integration.

### Phase 6: Accessibility, Polish, And Verification

**Goal**: The finished tools branch is stable across viewport sizes, keyboard
friendly, and safe within the existing app architecture.
**Depends on**: Phase 5
**Requirements**: UX-01, UX-03
**Success Criteria**:
1. `/tools` and `/tools/qr` remain legible and navigable on desktop and mobile.
2. Interactive controls have sensible accessible names and keyboard flow.
3. Motion and sheen effects support the holographic mood without making the UI
   hard to use.
4. `pnpm typecheck`, `pnpm lint`, and `pnpm build` all pass.
**Plans**: 2 plans

Plans:
- [x] 06-01: Polish responsive behavior, focus states, and reduced-motion-safe
  interactions.
- [x] 06-02: Run full verification, update planning state, and prepare the
  milestone summary.

### Phase 7: Style Prompt Index

**Goal**: The tools branch gains a practical style prompt library that feels
like an index, not a showcase, while keeping previews and prompt copying
reliable.
**Depends on**: Phase 6
**Requirements**: SP-01, SP-02, SP-03, SP-04, SP-05, SP-06, UX-04, UX-05
**Success Criteria**:
1. User can open `/tools/style-prompt` from the tools hub and browse a large
   local catalog without leaving the site.
2. Each visible family shows a reliable square preview image or safe fallback
   preview, rather than broken or empty canvases.
3. Opening a card reveals the preview, family description, and Chinese/English
   prompt copy actions in a single modal flow.
4. Template switching never leaves the user staring at unexplained blank prompt
   content; fallback behavior is explicit.
5. The page remains quieter than the styles it indexes, so browsing stays
   readable on desktop and mobile.
**Plans**: 2 plans

Plans:
- [x] 07-01: Build the local data registry, searchable gallery, and modal-based
  browsing flow for `Style Prompt Index`.
- [x] 07-02: Audit preview fidelity and polish the catalog UI so broken or
  misleading entries are filtered out before release.
