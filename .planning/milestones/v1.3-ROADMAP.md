# Roadmap: Word to Markdown

## Overview

Milestone v1.3 adds a hidden, browser-local Word to Markdown converter under
the existing tools branch. The first version focuses on conversion quality,
privacy-safe local processing, and downloadable Markdown/image assets before the
tool is promoted in `/tools`.

### Phase 8: Hidden Converter Foundation

**Goal**: Establish the hidden Word to Markdown route, local-only tool shell,
and dependency foundation without exposing the tool publicly.
**Depends on**: Phase 7
**Requirements**: W2M-01, W2M-02, W2M-03, W2M-05, W2M-12, W2M-14
**Success Criteria**:
1. User can directly open `/tools/word-to-markdown`.
2. `/tools` and the root homepage do not link to the hidden converter.
3. User can select or drag a `.docx` file into the page.
4. Invalid files, `.doc` files, and oversized files show clear errors.
5. The page clearly states that files stay in the browser and are not uploaded.
**Plans**: 2 plans

Plans:
- [x] 08-01: Add the hidden route, page shell, CSS module, and local-only user
  messaging.
- [x] 08-02: Add dependency foundation, file selection/drop handling, file
  validation, and explicit no-upload guardrails.

### Phase 9: Markdown Conversion Core

**Goal**: Convert `.docx` content locally into readable Markdown and expose
copy/download actions.
**Depends on**: Phase 8
**Requirements**: W2M-04, W2M-06, W2M-07, W2M-08, W2M-13
**Success Criteria**:
1. A valid `.docx` converts locally in the browser without server or Worker
   requests.
2. User can inspect the generated Markdown after conversion.
3. User can copy the generated Markdown.
4. User can download a `.md` file.
5. Conversion warnings and failures are visible in the UI.
**Plans**: 2 plans

Plans:
- [x] 09-01: Implement browser-local `.docx` to Markdown conversion and warning
  collection.
- [x] 09-02: Add Markdown result panel, copy action, `.md` download, and
  conversion status/error states.

### Phase 10: Image Assets And Verification

**Goal**: Extract embedded document images, package Markdown plus assets, and
verify the hidden converter end to end.
**Depends on**: Phase 9
**Requirements**: W2M-09, W2M-10, W2M-11
**Success Criteria**:
1. Embedded `.docx` images are extracted into generated asset files.
2. Markdown references extracted images with stable relative paths such as
   `./assets/image-1.png`.
3. User can download a zip containing the Markdown file and `assets/` directory.
4. `pnpm typecheck`, `pnpm lint`, and `pnpm build` pass.
5. Manual UAT covers valid `.docx`, image documents, invalid file types,
   oversize rejection, copy, `.md` download, and zip download.
**Plans**: 2 plans

Plans:
- [x] 10-01: Add image extraction, stable asset naming, and zip export.
- [x] 10-02: Run verification/UAT, update planning state, and decide whether
  the tool is ready for future public promotion.
