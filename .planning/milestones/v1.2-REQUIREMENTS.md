# Requirements: Tools Branch Evolution

**Defined:** 2026-05-02
**Core Value:** Distinct visual identity should never come at the cost of
instantly understandable navigation or tool usefulness.

## v1.2 Requirements

Requirements for milestone v1.2, which adds a local UI style prompt library
inside `/tools`.

### Style Prompt Index

- [x] **SP-01**: Visitor can open `/tools/style-prompt` from the tools hub and
  understand the page as a prompt index rather than a generic landing page.
- [x] **SP-02**: Visitor can browse the catalog by category and narrow it with
  keyword search.
- [x] **SP-03**: Each visible family presents a stable square preview image or a
  safe fallback preview, rather than a broken blank state.
- [x] **SP-04**: Opening a card shows the preview, family context, and direct
  copy actions for Chinese and English prompt text.
- [x] **SP-05**: When a template has no standalone prompt, the interface makes
  the family-prompt fallback explicit instead of appearing empty or broken.
- [x] **SP-06**: Families that cannot present a usable preview are removed from
  the published catalog.

### UX And Safety

- [x] **UX-04**: The page stays visually restrained enough that prompt content
  and preview browsing remain the primary focus.
- [x] **UX-05**: The tool runs from local static assets only, with no auth,
  uploads, cloud storage, or third-party runtime execution.

## v1.1 Requirements

Requirements for milestone v1.1, which introduces the `/tools` branch and the
first public utility.

### Tools Hub

- [x] **HUB-01**: Visitor can open `/tools` and immediately understand it is
  the site's tools hub.
- [x] **HUB-02**: `/tools` presents tool cards with clear names, one-line
  purposes, and visible availability states.
- [x] **HUB-03**: `/tools` uses the holographic film visual language while
  keeping labels, descriptions, and primary actions easy to read.
- [x] **HUB-04**: Visitor can open the live QR tool from the hub in one click.

### QR Studio

- [x] **QR-01**: User can enter plain text or a URL and generate a QR code
  without leaving the page.
- [x] **QR-02**: User can adjust QR size, quiet-zone margin, foreground color,
  and background color before download.
- [x] **QR-03**: User can download the generated QR code as a PNG file.
- [x] **QR-04**: User can download the generated QR code as an SVG file.
- [x] **QR-05**: User can quickly clear or replace the current input without
  reloading the page.

### Portal Integration

- [x] **PORT-01**: The root portal replaces one sealed panel with a real
  `Tools` entrance that links to `/tools`.
- [x] **PORT-02**: The new `Tools` panel preserves the comic grid's visual
  balance on desktop and mobile.
- [x] **PORT-03**: One sealed future panel remains so the homepage still feels
  like an expanding universe rather than a completed sitemap.

### UX And Safety

- [x] **UX-01**: Interactive controls on `/tools` and `/tools/qr` have
  accessible names and usable keyboard flow.
- [x] **UX-02**: The QR tool remains frontend-only and does not require auth,
  uploads, storage, or backend execution.
- [x] **UX-03**: Implementation passes `pnpm typecheck`, `pnpm lint`, and
  `pnpm build`.

## Later Milestone Ideas

Deferred beyond v1.2.

### Future Tools

- **FUT-01**: Add more safe public tools such as JSON, Markdown, color, or time
  utilities.
- **FUT-02**: Add protected or storage-backed tools once auth, abuse limits,
  and data lifecycle are defined.
- **FUT-03**: Decide whether future tools need history, favorites, or cloud
  persistence.

## Out of Scope

Explicitly excluded for this milestone.

| Feature | Reason |
|---------|--------|
| Word-to-Markdown uploads | Requires file handling, storage, and abuse-boundary decisions that are not needed for QR Studio |
| API sender / proxy tooling | Needs backend proxying and abuse controls; too risky for this public-first milestone |
| Browser code execution | Not worth the security and UX risk for the first tools release |
| QR history, auth, or cloud persistence | The first tool should stay instant and public-safe, with no storage burden |
| Changes to Game or Photos behavior | This milestone expands Tools and the homepage entrance map only |
| Replacing the final sealed homepage panel | The homepage should still communicate future expansion |
