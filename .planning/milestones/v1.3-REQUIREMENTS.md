# Requirements: Word to Markdown

**Defined:** 2026-05-10
**Core Value:** Distinct visual identity should never come at the cost of
instantly understandable navigation or tool usefulness.

## v1.3 Requirements

Requirements for milestone v1.3, which adds a hidden browser-local Word to
Markdown converter under `/tools`.

### Hidden Route

- [x] **W2M-01**: User can open the hidden route `/tools/word-to-markdown`.
- [x] **W2M-02**: The Word to Markdown tool is not exposed from the `/tools`
  hub or root homepage during this milestone.

### Local Conversion

- [x] **W2M-03**: User can select or drag in one `.docx` file.
- [x] **W2M-04**: The tool converts the file locally in the browser without
  upload, persistent storage, server-side execution, or Worker API calls.
- [x] **W2M-05**: The tool rejects `.doc`, non-Word files, and files over the
  configured size limit with clear user-facing errors.

### Markdown Output

- [x] **W2M-06**: User can view the generated Markdown after conversion.
- [x] **W2M-07**: User can copy the generated Markdown.
- [x] **W2M-08**: User can download the generated Markdown as a `.md` file.

### Image Assets

- [x] **W2M-09**: If the `.docx` contains embedded images, the tool extracts
  image assets during conversion.
- [x] **W2M-10**: User can download a zip containing the Markdown file and an
  `assets/` image directory.
- [x] **W2M-11**: Generated Markdown uses stable relative image paths such as
  `./assets/image-1.png`.

### Safety And UX

- [x] **W2M-12**: The page clearly states that files are processed locally in
  the user's browser.
- [x] **W2M-13**: Conversion status, warnings, and failure reasons are shown in
  the UI.
- [x] **W2M-14**: The milestone does not add cloud history, R2 storage, account
  features, uploads, or server-side document processing.

## Future Requirements

Deferred beyond v1.3.

### Public Promotion

- **W2M-FUT-01**: Expose Word to Markdown from the `/tools` hub once conversion
  quality passes UAT with representative documents.
- **W2M-FUT-02**: Add public-facing copy and polish for the tool card.

### Storage And History

- **W2M-FUT-03**: Add optional conversion history after auth, privacy, retention,
  and R2 cost boundaries are defined.
- **W2M-FUT-04**: Allow users to delete stored conversion records and files.

### Format Expansion

- **W2M-FUT-05**: Evaluate legacy `.doc` conversion through a controlled server
  path if the use case is worth the added security and operations burden.
- **W2M-FUT-06**: Evaluate PDF or other document conversions in a separate
  milestone.

## Out of Scope

Explicitly excluded for this milestone.

| Feature | Reason |
|---------|--------|
| `.doc` legacy Word support | Old binary Word files are not equivalent to `.docx` and likely require server-side tooling |
| Cloud upload or Worker conversion | v1.3 is local-only to avoid privacy, retention, and abuse-boundary work |
| R2-backed file history | Persistent document storage needs auth, privacy policy, deletion, file-size limits, and cost boundaries |
| Public `/tools` card | The tool should stay hidden until conversion quality is verified |
| Layout-perfect Word reproduction | Markdown cannot preserve all Word layout semantics; output should be semantic Markdown |
| OCR or scanned document support | Requires a different processing model and substantially more complexity |
| Game or Photos changes | This milestone only adds a hidden tools-branch converter |
