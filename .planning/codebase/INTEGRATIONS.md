# Codebase Map: Integrations

Date: 2026-05-01

## Summary

The site integrates with Steam Web API, two Cloudflare Worker APIs, Cloudflare R2,
Cloudflare Workers/OpenNext deployment, GitHub Actions, browser localStorage,
and Steam CDN image hosts. Secrets are referenced by environment variable name
only in this document.

## Domain Routing

- `middleware.ts` rewrites only root requests based on the `Host` header.
- `game.*` root requests are rewritten to `/game`.
- `photo.*` root requests are rewritten to `/photos`.
- Non-root paths are passed through with `NextResponse.next()`.
- Main domain root stays on `app/page.tsx`.

## Steam Web API

- `scripts/fetch-steam-data.js` calls Steam APIs from `https://api.steampowered.com`.
- Player summary endpoint:
  - `ISteamUser/GetPlayerSummaries/v2`
- Owned games endpoint:
  - `IPlayerService/GetOwnedGames/v1`
- Recently played endpoint:
  - `IPlayerService/GetRecentlyPlayedGames/v1`
- Required environment variables:
  - `STEAM_API_KEY`
  - `STEAM_ID`
- The script merges `playtime_2weeks` from recent games into owned games.

## Game API Worker

- Source: `workers/game-api/src/index.ts`.
- Worker name: `mays-game-api` in `workers/game-api/wrangler.toml`.
- R2 binding: `GAME_BUCKET`.
- R2 bucket name: `my-games`.
- Public read endpoints:
  - `GET /api/steam-games`
  - `GET /api/steam-games.json`
- Health endpoint:
  - `GET /health`
- Upload endpoint:
  - `POST /api/upload`
- Upload authentication uses bearer token from `UPLOAD_SECRET`.
- CORS origin default is configured by `CORS_ORIGIN`.

## Photo API Worker

- Source: `workers/photo-api/src/index.ts`.
- Worker name: `mays-photo-api` in `workers/photo-api/wrangler.toml`.
- R2 binding: `PHOTO_BUCKET`.
- R2 bucket name: `my-photos`.
- Auth endpoint:
  - `POST /api/auth`
- Group endpoints:
  - `GET /api/groups`
  - `POST /api/groups`
- Photo object endpoints:
  - `POST /api/upload`
  - `GET /api/image/:groupId/:photoId`
  - `DELETE /api/image/:groupId/:photoId`
- Password auth uses `AUTH_PASSWORD`.
- Token signing uses `JWT_SECRET` when present, otherwise a derived fallback from `AUTH_PASSWORD`.
- CORS origin default is configured by `CORS_ORIGIN`.

## Frontend API Clients

- Game client default API URL is in `lib/game.ts` as `GAME_API_URL`.
- Game API override uses `NEXT_PUBLIC_GAME_API_URL`.
- Steam client store reads Worker data first, then falls back to `/data/steam-games.json` in `stores/steam-store.ts`.
- Photo client default API URL is in `lib/photo-api.ts`.
- Photo API override uses `NEXT_PUBLIC_PHOTO_API_URL`.
- Photo API functions include `authenticate`, `fetchGroups`, `saveGroups`, `uploadPhoto`, `fetchPhotoBlob`, and `deletePhoto`.

## Browser Storage

- Photo groups are persisted to browser localStorage.
- Storage key is `mays_photo_groups` in `lib/photo.ts`.
- Auth token is persisted to browser localStorage.
- Token key is `mays_photo_token` in `lib/photo-api.ts`.
- `stores/photo-store.ts` writes localStorage before attempting Worker persistence for group mutations.

## Cloudflare R2 Objects

- Game Worker stores Steam data at `steam-games.json`.
- Photo Worker stores group metadata at `metadata/groups.json`.
- Photo Worker stores image binaries under `images/${groupId}/${photoId}`.
- Photo image responses copy R2 HTTP metadata and set long-lived cache headers.

## GitHub Actions

- Steam data workflow is `.github/workflows/deploy.yml`.
- Steam workflow runs daily at UTC 02:00 and through `workflow_dispatch`.
- Steam workflow writes `public/data/steam-games.json` and may commit the snapshot.
- Frontend deploy workflow is `.github/workflows/deploy-frontend.yml`.
- Frontend deploy workflow runs on pushes to `main` and through `workflow_dispatch`.
- Frontend deployment uses `pnpm run deploy`.

## GitHub And Cloudflare Configuration

- GitHub Secrets referenced by workflows:
  - `STEAM_API_KEY`
  - `STEAM_ID`
  - `GAME_UPLOAD_SECRET`
  - `CLOUDFLARE_API_TOKEN`
  - `CLOUDFLARE_ACCOUNT_ID`
- GitHub Variables referenced by frontend deployment:
  - `NEXT_PUBLIC_GAME_API_URL`
  - `NEXT_PUBLIC_PHOTO_API_URL`
- Worker secrets referenced in docs and code:
  - `UPLOAD_SECRET`
  - `AUTH_PASSWORD`
  - `JWT_SECRET`

## External Image And Store Links

- Steam Store links are generated in `components/game/GameCard.tsx`, `components/game/FeaturedGame.tsx`, and `components/game/GameHome.tsx`.
- Steam header images are generated by `getGameHeaderImage` in `lib/game.ts`.
- Steam capsule images are generated by `getGameCapsuleImage` in `lib/game.ts`.
- Next image remote patterns are configured in `next.config.mjs`.

## Operational Scripts

- `scripts/fetch-steam-data.js` fetches Steam data, uploads to Game API Worker, and writes the local snapshot.
- `scripts/fetch-city-coordinates.js` fetches city coordinates from OpenStreetMap Nominatim and writes a local JSON output.
- The city coordinate script is not wired into `package.json` scripts.
